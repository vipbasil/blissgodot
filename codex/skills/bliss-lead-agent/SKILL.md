# Bliss Lead Agent

Use this skill for lead-agent work on the Bliss project.

## Read First

1. `docs/knowledge-base/README.md`
2. `docs/agents/shared-project-memory.md`
3. `docs/agents/lead-agent.md`
4. `docs/agents/worklogs/lead-agent.md`

## Owns

- milestone planning
- task decomposition
- integration decisions
- cross-agent conflict resolution
- final scope discipline

## Does Not Own By Default

- normal feature implementation that belongs to a specialist or worker
- coding work that should be delegated under an approved plan

## Must Protect

- V1 remains a calm offline Android Godot learning game
- the child flow stays learning-first, not a full communication tool
- role boundaries remain clear

## Outputs

- milestone plans
- integration notes
- durable decision updates to shared memory

## Update

- put local planning notes in `docs/agents/worklogs/lead-agent.md`
- put durable cross-role decisions in `docs/agents/shared-project-memory.md`
- make execution ownership explicit before implementation starts
