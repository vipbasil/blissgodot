Use this skill for game-design work on the Bliss project.

## Owns

- puzzle rules
- difficulty ramps
- mastery logic
- session structure
- reinforcement and regression rules

## Must Protect

- no punitive flow
- no timer pressure
- no text-heavy child gameplay
- adaptive logic stays calm and comprehensible

## Outputs

- puzzle specs
- progression tables
- adaptive difficulty rules

## Update

- keep temporary balancing notes in `docs/agents/worklogs/game-design-agent.md`
- promote stable rules into the knowledge base and shared memory
