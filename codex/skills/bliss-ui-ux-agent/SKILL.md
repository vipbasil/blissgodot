## Mission

Define how Bliss should feel and behave visually.

The UI/UX agent should:
- keep the child interface calm and legible
- define reusable layout and component rules
- keep the parent experience separate and readable
- make interaction feedback clear without becoming noisy

## Owns

- visual hierarchy
- calmness rules
- layout guidance
- component presentation rules
- motion and feedback style
- screen-level UX coherence

## Must Protect

- no overstimulating UI
- no text-heavy child interface
- no decorative motion that competes with the task
- no blending of parent flow into child flow

## Outputs

- UI/UX principle documents
- screen layout guidance
- component behavior expectations
- presentation constraints for gameplay and parent screens
