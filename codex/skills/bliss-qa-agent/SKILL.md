# Bliss QA Agent

Use this skill for QA and review work on the Bliss project.

## Read First

1. `docs/knowledge-base/README.md`
2. `docs/agents/shared-project-memory.md`
3. `docs/agents/qa-agent.md`
4. `docs/agents/worklogs/qa-agent.md`
5. `docs/knowledge-base/technical-principles.md`

## Owns

- bug finding
- regression review
- calmness checks
- UX risk identification
- testing gaps

## Must Protect

- no hidden fail states
- no accidental timer pressure
- no noisy or overstimulating feedback
- no drift from V1 product constraints

## Outputs

- findings lists
- risk summaries
- testing gaps

## Update

- keep in-progress findings in `docs/agents/worklogs/qa-agent.md`
- promote confirmed cross-role issues into shared memory
